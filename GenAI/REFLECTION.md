# Reflection

Throughout the DSIP project, AI served as a development assistant rather than a replacement for my own understanding. It was most helpful during the implementation of the ADO.NET repository and the ASP.NET Core Web API. A specific example was building the `ShipmentRepository` class. AI helped generate the initial structure for database operations using parameterized SQL commands and proper `using` blocks for resource management. This reduced repetitive coding effort and allowed me to focus on the application's logic, architecture, and integration between the API and the database.

Although AI accelerated development, I did not accept every suggestion without review. In several cases, the generated code did not fully satisfy the project requirements. For example, AI initially suggested naming the SQL view `vw_ShipmentDashboard`, whereas the assessment explicitly required the view to be named `ShipmentDashboard`. In another instance, the Python `export_csv()` function was placed incorrectly, resulting in an `UnboundLocalError`. Rather than assuming the generated code was correct, I analyzed the errors, compared the implementation with the project specification, identified the root cause, and modified the code accordingly. These experiences reinforced the importance of understanding the generated code before using it.

Every AI-generated solution was verified before becoming part of the project. I compared the implementation against the assessment requirements, compiled the code, and tested each module independently. SQL scripts were executed in SQL Server to verify stored procedures, views, indexes, and seed data. Web API endpoints were tested using Swagger to confirm correct HTTP status codes, validation, and JSON responses. The frontend was tested to ensure that AJAX operations updated the interface without page reloads, and the Python analytics script was validated by comparing the generated report and exported CSV against the actual data stored in the database.

This project changed the way I use AI during development. Instead of expecting complete solutions, I found the greatest value in using AI to generate initial implementations, explain unfamiliar concepts, and accelerate repetitive tasks. Going forward, I will continue using AI as a productivity tool while relying on testing, debugging, and my understanding of the requirements to determine whether the generated code is correct. This approach allows me to benefit from AI without compromising software quality or correctness.
